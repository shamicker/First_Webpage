HTML Basics

HTML Components and Syntax

The web's primary language is HTML. Items on the web can be made up of any kind of content: text, markup (what it looks like), references to documents, and links. Like any language, HTML is made up of smaller components and syntax (grammar laws). There are tags, elements, attributes, and values.

A tag is the equivalent of a "word". A tag has < and > around it.

An element is usually made up of a beginning tag and an end tag. Sometimes there is no closing tag, called a void tag since there is no actual content.

There is also an attribute, which is a specified characteristic of a tag, and a value is the description of the attribute. As in <tag tag's-attribute="value">content</tag>.

HTML Structure

Structured HTML is crucial in order to create an efficient page and increase productivity. It is the outline of your document, and creates some of the hierarchy in which CSS operates. If an HTML document is well-structured, adding CSS should be a breeze.

There are 2 types of elements - inline and block elements. An inline element is where an element just continues in line with what comes before and after the element Block elements actually separate themselves a bit, and create a little invisible barrier or block around themselves. Block elements also affect layout and style, whereas inline elements do not.

CSS Basics

CSS Components and Syntax

CSS stands for Cascading Style Sheets. 'Cascading' because of the cascading hierarchy which falls down from parent to child levels. The least powerful level of cascade is the least specific one, often firstly mentioned. In other words, it's the 'blanket statement' rule that applies to everything that does not have more specific rules.

Selectors make Declarations! This is everything within the curly brackets { }. A declaration is made up of a Property with a Value. The main selectors are:

Tag - (selector { }) - a sort of blanket statement saying all of this [whatever is chosen] element is styled this [whatever is specified] way.

Class - (.selector { }) - is very versatile and is very common.

Pseudo-class - (:selector { }) - can be positional or relational. For instance, '3n' would be 'every third'; 'hover' would be 'when hovered over'.

ID - (#selector { }) - is the strongest and over-rules all other selectors. Therefore it is not used too frequently.

For more on selectors, check out Mozilla CSS Selectors (I don't know why it's in Spanish right now since English is the selected language!) and Mozilla CSS References.

CSS Style

Code is used to communicate with a computer, but style is used to communicate with others (or your future self). 3 major reasons to have good style:

  • to write more efficiently
  • to understand already-written code more efficiently
  • to communicate in a common style

Commenting in your code can be important to understanding your style. A comment or note-to-self in HTML is denoted by <!--text-->. In CSS it is /*text*/. You can leave a short description, as in /*this section is here*/, or a longer note or reminder, as in /*to do: make x changes here!*/.

As it is said, apparently:

  1. Beginners comment on what they're doing.
  2. Experienced comment on how they're doing something.
  3. Experts comments on why they're doing something.

If you have good style, it already says what you're doing.

Semantics

It always comes down to semantics (meaning of content)! Proper use is needed in programming, where being specific and precise is essential to good coding (unlike current English skills). Semantics is key.

For example, to emphasize a key word, one could use <b> or <em> or <strong>. But it all depends on what your goal is. To just bold or italicize is just that. However, if you really want to emphasize the meaning, then you would use <strong>.

Python Basics

Components and Syntax

Unlike HTML (and CSS), Python is actually an interpreter! The code we write in Python language is the human side of the instructions, as precise as we can get. It's put into the python program and the program translates our version of Python into a language that the computer can understand, and then the computer executes those instructions. If using in Sublime Text, just make sure to save and then hit cmd-B (on a Mac) for it to execute at the bottom of the screen!

Much like Microsoft Excel, Python is more of a mathematical language than HTML is. A human assigns names to variables, creates statements and procedures (functions), and then these expressions are executed by the computer.

A variable is a name given to a number or string that is more memorable to the viewer (human). For instance, instead of having an expression of 299792458, which is a seemingly random number, one would give it a name of speed_of_light.

Sometimes, numbers without a variable magic numbers. It's best to reduce the amount of "magic numbers" in our code. In this way, one not only minimizes the margin of error, but also creates code that is equally readable by the computer and humans.

A statement is simply the "official" assigning, if you will, of a name/variable. Python won't read minds and requires explicit statements. For the above example, an assignment statement would look like: speed_of_light = 299792458.

A procedure is also known as a function (again, think Excel). A method is a built-in procedure in Python. For example, .find is a method: Python already knows what to do with it.

An expression is something that has a value (not necessarily numerical); is in some way expressed. It can be numerical, alphabetical, a string, or something that will be defined later.

A string is a value that has quotation marks around it, representing a non-numerical value (although it can contain numbers). It can be a single character or a string of them.

In Python, the = sign does not signify equality, but rather is treated more like an arrow; it's an assignment. So 2 + 3 = 5 is very different from my_variable = 5.

Decimals must be forced, in Python. By default, Python will not give decimals, yet it won't round either! It just provides the whole number. If you want decimals, there must be a decimal point somewhere in your variables or numbers.

Indexing

To index a string is to find the first position within the string, of whatever you're looking for. The result will be a numerical value. The very first position of a string is always 0, so the second character's position is always 1. To start at counting (backwards) from the end, one uses negative numbers, starting with -1 being the very last character position.

For example: "udacity"[0] is where, for the string "udacity", we are indexing (denoted by [ and ]) the first character (by using 0 since all indexing sequences start with 0). The result is u. A couple others:

  1. "udacity"[3] = c
  2. "udacity"[-1] = y

Although I easily could, since all this is new to me, I will not go into further detail! I will, however, redirect you to a Python Reference sheet and some more complicated procedures

**Note that there is a difference between <string>[0] and <string>[0:]. If there are any characters in the <string>, then there is no problem: the first results in the first character of the string, while the second expression results in the whole string. However, if the string is empty, the first expression will error out because there are no characters and therefore cannot give the first character. On the other hand, the second expression won't cause an error because the result will just display the empty string.**

Finding

As previously mentioned, find is a pre-defined method; Python can find a character or a string within another string, and it's basically the same as the Excel function. Like indexing, find will numerically return the first position of what you're looking for, but anything not found will execute as -1.

Syntax for find is <string>.find(<string>).

Other Concepts

DOM Tree

DOM stands for Document Object Model and is an all-language concept. It is the breaking down of a page into smaller and smaller parts, eventually ending up with a page's most basic components (much like finding the prime factors of a number).

For Later Learning

Some concepts I've stumbled upon to keep in mind:

  • Overflow (CSS) maintains text within a contained area using a scroll bar to view the "overflowing" text.
  • Frameworks are sets of (CSS) classes that control layout and are standard-compliant. Many are pre-prepared, such as 'Bootstrap' or 'Foundation'. Most contain a grid and maybe some basic (design-oriented) JavaScript.
